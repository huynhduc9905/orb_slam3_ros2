# Standalone Nix DevShell for orb_slam3_ros2

**Date:** 2026-07-20  
**Goal:** Add a self-contained `flake.nix` inside `orb_slam3_ros2/` so the repo can be cloned onto any NixOS workstation and built with `nix develop` + `colcon build` — independent of the large `~/robot` workspace.

---

## Problem

The current build environment lives in `~/robot/flake.nix`, which pulls in a massive closure: navigation2, slam-toolbox, cartographer, realsense, rplidar, Gazebo sim deps, etc. None of that is needed by the ORB-SLAM3 packages. This makes it impractical to transfer the project to another workstation without also transferring the entire robot workspace.

---

## Approach

**Approach B: Fully self-contained `flake.nix` inside `orb_slam3_ros2/`.**

A `flake.nix` at the repo root that:
- Has its own `nix-ros-overlay` input — no dependency on `~/robot/flake.nix`
- Declares only the deps needed by the 6 packages in this repo
- Configures `ros.cachix.org` for fast binary downloads on the workstation
- Provides a `devShells.default` with the same workflow: `nix develop` → `colcon build`

---

```
orb_slam3_ros2/
├── flake.nix        ← new
├── flake.lock       ← generated by `nix flake update`
```

## shellHook

```bash
unset PROMPT_COMMAND

if [ -f install/setup.bash ]; then
  source install/setup.bash
fi

export ROS_DOMAIN_ID=${ROS_DOMAIN_ID:-0}
export RMW_IMPLEMENTATION=${RMW_IMPLEMENTATION:-rmw_fastrtps_cpp}

if command -v register-python-argcomplete >/dev/null 2>&1; then
  eval "$(register-python-argcomplete --shell bash ros2 2>/dev/null)" || true
  eval "$(register-python-argcomplete --shell bash colcon 2>/dev/null)" || true
fi

echo "orb_slam3_ros2 standalone devshell active (ROS 2 Kilted)"
```

## Git Transfer Workflow

1. Commit `flake.nix` + `flake.lock` (with any pending WIP changes as a temp commit if needed)
2. `git push origin <branch>`
3. On workstation: `git clone git@github.com:huynhduc9905/orb_slam3_ros2.git`
4. `cd orb_slam3_ros2 && git submodule update --init`
5. `nix develop` — downloads prebuilt ROS binaries from ros.cachix.org
6. `colcon build --packages-select orb_slam3_vendor orb_slam3_msgs orb_slam3_wrapper orb_lidar_mapper orb_slam_dashboard orb_slam_bringup --cmake-args -DBUILD_TESTING=ON -DCMAKE_BUILD_TYPE=Release --parallel-workers $(nproc)`

Note: `--parallel-workers 1` in the README is a workaround for the resource-constrained current machine. On the faster workstation, use `$(nproc)` or a higher value.

---

## Success Criteria

- `nix develop` enters the shell without errors on a fresh NixOS workstation
- `colcon build` succeeds for all 6 packages
- `ros2 topic list` works inside the shell after sourcing `install/setup.bash`
- No dependency on `~/robot/` or any other path outside the repo
